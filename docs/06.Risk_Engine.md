# Risk Engine — Knowledge Transfer

## What is it?

The Risk Engine is a **deterministic validation gate** that sits between the Options AI Agent and the database. Every AI-generated recommendation passes through this service before being saved. It enforces hard rules that the LLM cannot override — position limits, DTE constraints, probability thresholds, and risk/reward ratios.

**Philosophy**: AI suggests, rules enforce. The LLM might recommend a risky trade; the risk engine blocks it.

---

## High-Level Flow

```
Options AI Agent
    │
    │ POST /api/risk/validate (recommendation JSON)
    │
    ▼
Risk Engine
    │
    ├─ Load config from Postgres (max_position_pct, dte_range, ...)
    ├─ Check existing positions (portfolio exposure)
    ├─ Check pending recommendations (concentration)
    ├─ Validate DTE, probability, risk/reward
    │
    ▼
Response: { approved: true/false, flags: [...], risk_notes: "..." }
```

## Validation Rules

### Rule 1: Position Size Limit

[**Code: main.py:57-70**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L57)

```python
max_position_pct = float(config.get("max_position_pct", 5))
total_portfolio = sum(p["market_value"] for p in positions) or 100000
symbol_exposure = sum(p["market_value"] for p in positions if p["symbol"] == rec.get("symbol", ""))
new_pct = ((symbol_exposure + capital_required) / total_portfolio) * 100

if new_pct > max_position_pct:
    flags.append(f"Position would exceed {max_position_pct}% limit ({new_pct:.1f}%)")
```

**What it checks**: Would this trade put more than X% of the portfolio into a single symbol?

**Default**: 5% max per position. If you have a $100K portfolio and already have $4,000 in NVDA, a trade requiring $2,000 capital would bring total to $6,000 (6%) → **flagged**.

**Fallback**: If no positions exist, assumes $100,000 portfolio (prevents division by zero).

---

### Rule 2: DTE Range

[**Code: main.py:73-84**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L73)

```python
dte_range = config.get("dte_range", {"min": 14, "max": 60})
for leg in rec.get("legs", []):
    expiry = datetime.strptime(leg["expiry"], "%Y-%m-%d")
    dte = (expiry - datetime.utcnow()).days
    if dte < dte_range.get("min", 14):
        flags.append(f"DTE {dte} below minimum {dte_range['min']}")
    if dte > dte_range.get("max", 60):
        flags.append(f"DTE {dte} above maximum {dte_range['max']}")
```

**What it checks**: Are the option expirations within the configured DTE window?

**Default**: 14-60 days. Options too close to expiration (< 14 DTE) have gamma risk (rapid price swings). Options too far out (> 60 DTE) tie up capital and have more uncertainty.

**Per-leg check**: Each leg of the spread is validated independently. A calendar spread with one leg at 10 DTE and another at 45 DTE would flag the 10 DTE leg.

---

### Rule 3: Concentration Check

[**Code: main.py:87-89**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L87)

```python
open_count = await get_pending_recommendations(rec.get("symbol", ""))
if open_count >= 3:
    flags.append(f"Already {open_count} active positions on {rec.get('symbol')}")
```

**What it checks**: How many "taken" (active) recommendations exist for this symbol?

**Threshold**: 3 or more → flagged. This prevents over-concentration — if you already have 3 open options trades on NVDA, adding a 4th increases correlated risk.

---

### Rule 4: Risk/Reward Ratio

[**Code: main.py:92-96**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L92)

```python
if max_loss > 0 and max_profit > 0:
    rr_ratio = max_profit / max_loss
    if rr_ratio < 0.15:
        flags.append(f"Poor risk/reward ratio: {rr_ratio:.2f}")
```

**What it checks**: Is the maximum profit at least 15% of the maximum loss?

**Example**:
- Max profit $420, max loss $1,580 → ratio = 0.27 → OK
- Max profit $50, max loss $1,950 → ratio = 0.026 → **flagged** (risking $1,950 to make $50 is terrible)

---

### Rule 5: Probability Threshold

[**Code: main.py:99-101**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L99)

```python
prob = rec.get("probability_estimate", 0)
if prob < 0.5:
    flags.append(f"Low probability: {prob:.0%}")
```

**What it checks**: Does the recommended trade have at least 50% probability of profit?

**Why 50%?** For income strategies (selling premium), you want high-probability trades. Anything below 50% is essentially a coin flip with defined risk.

---

### Decision Logic

[**Code: main.py:103-107**](https://github.com/utkarshmahar/stocks/blob/main/services/risk-engine/main.py#L103)

```python
approved = len(flags) == 0
risk_notes = "; ".join(flags) if flags else "All risk checks passed"
return {"approved": approved, "risk_notes": risk_notes, "flags": flags}
```

**Binary decision**: If ANY flag is raised, the recommendation is **not approved**. There's no severity weighting — any violation is a hard block.

The `risk_notes` string joins all flags with semicolons for display in the frontend.

## Configuration (from Postgres `agent_config` table)

| Key | Default | Description |
|-----|---------|-------------|
| `max_position_pct` | 5 | Max single-position size as % of portfolio |
| `dte_range` | {"min": 14, "max": 60} | Acceptable days-to-expiration window |
| `delta_range` | {"min": 0.15, "max": 0.35} | Delta range for short legs (not yet checked in risk engine) |
| `risk_tolerance` | "moderate" | Not yet used in risk checks (future enhancement) |
| `max_sector_pct` | 20 | Max sector exposure (not yet implemented) |

These are configurable from the Settings page in the frontend.
