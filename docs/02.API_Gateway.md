# API Gateway — Knowledge Transfer

## What is it?

The API Gateway is the **single entry point** for the React frontend to communicate with all backend services. It's a FastAPI application that provides:

1. **REST endpoints** — CRUD for watchlist, recommendations, portfolio, config, quotes
2. **WebSocket endpoint** — Streams live options data to the browser in real-time
3. **InfluxDB proxy** — Queries time-series data and returns clean JSON to the frontend
4. **Service health aggregator** — Pings all microservices and reports their status

The frontend **never** talks directly to InfluxDB, Postgres, or other microservices. Everything goes through this gateway.

---

## High-Level Flow

```
Frontend (React)
    │
    ├─ REST (HTTP) ──→ API Gateway ──→ PostgreSQL (CRUD)
    │                       │
    │                       ├──→ InfluxDB (quotes, options chain)
    │                       │
    │                       └──→ Other services (health checks)
    │
    └─ WebSocket ────→ API Gateway ──→ Redis pub/sub (live options updates)
```

## Technology Deep Dive: FastAPI

### What is FastAPI?

[FastAPI](https://fastapi.tiangolo.com/) is a modern Python web framework built on top of **Starlette** (ASGI) and **Pydantic** (data validation). Key properties:

- **Async-native**: Built on Python's `async/await`, it handles concurrent requests without threads
- **ASGI** (Asynchronous Server Gateway Interface): Unlike WSGI (Flask/Django), ASGI supports WebSockets, long-lived connections, and non-blocking I/O
- **Automatic OpenAPI docs**: Generates Swagger UI at `/docs` and ReDoc at `/redoc`
- **Dependency injection**: The `Depends()` system provides database sessions, auth tokens, etc.

### Why FastAPI for this project?

| Requirement | FastAPI Feature |
|------------|----------------|
| WebSocket streaming | Native WebSocket support via ASGI |
| Non-blocking DB queries | `async/await` with SQLAlchemy async sessions |
| Auto-documentation | Swagger UI at `/docs` |
| Input validation | Pydantic models for request/response |
| Lightweight | Small memory footprint (critical for Raspberry Pi) |

### Uvicorn: The ASGI Server

[**Code: Dockerfile CMD**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/Dockerfile#L12)

```dockerfile
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Uvicorn** is the ASGI server that actually runs FastAPI. Think of it like:
- **Uvicorn** = the web server (handles TCP connections, HTTP parsing)
- **FastAPI** = the application framework (routing, request handling, response formatting)

Uvicorn uses `uvloop` (a fast C-based event loop) under the hood for maximum async performance.

```python
app.add_middleware(

    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],

)
```

**Why this matters**: The frontend runs on port 3001, the API on port 8000. Browsers block cross-origin requests by default (Same-Origin Policy). CORS headers tell the browser "it's OK for port 3001 to call port 8000."

`allow_origins=["*"]` allows any origin. In production, you'd restrict this to your domain.

### 2. Watchlist Management (CRUD)

[**Code: main.py:31-65**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/main.py#L31)

```python
@app.get("/api/watchlist")     # List all tracked symbols
@app.post("/api/watchlist")    # Add a symbol
@app.delete("/api/watchlist/{symbol}")  # Remove (soft-delete) a symbol
```

**How it works**:
- `GET`: Reads all rows from the `watchlist` table in Postgres
- `POST`: Inserts a new symbol; uses `ON CONFLICT DO UPDATE SET active = TRUE` (upsert) so re-adding a removed symbol reactivates it
- `DELETE`: Sets `active = FALSE` (soft delete). The ingestion service only polls `WHERE active = TRUE`

**The dependency injection pattern**:
```python
async def get_watchlist(db: AsyncSession = Depends(get_db)):
```

`Depends(get_db)` tells FastAPI:
1. Before running this endpoint, call `get_db()` to get a database session
2. Inject that session as the `db` parameter
3. After the endpoint returns, close the session automatically

This is Python's version of middleware/DI — it's declarative, testable, and handles cleanup.

### 3. Live Quotes from InfluxDB

[**Code: main.py:70-95**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/main.py#L70)

```python
@app.get("/api/quote/{symbol}")
async def get_quote(symbol: str):
    query = f'''
    from(bucket: "{settings.influxdb_bucket}")
      |> range(start: -5m)
      |> filter(fn: (r) => r._measurement == "stock_quote" and r.symbol == "{symbol.upper()}")
      |> last()
      |> pivot(rowKey: ["_time"], columnKey: ["_field"], valueColumn: "_value")
    '''
```

**Flux query language explained**:
- `from(bucket: "options_db")` — Which InfluxDB bucket to query
- `|> range(start: -5m)` — Only look at data from the last 5 minutes
- `|> filter(...)` — Select only `stock_quote` measurement for the given symbol
- `|> last()` — Get the most recent data point
- `|> pivot(...)` — Convert InfluxDB's columnar format into a flat row (so `price`, `bid`, `ask` become columns instead of separate rows)

**Why -5m range?** The ingestion service polls every 30 seconds. A 5-minute window ensures we find data even if a few polls failed. `last()` ensures we get the most recent point within that window.

### 6. WebSocket: Live Options Streaming

[**Code: main.py:262-278**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/main.py#L262)

```python
@app.websocket("/ws/options")
async def websocket_options(websocket: WebSocket):
    await websocket.accept()
    redis = await get_redis()
    pubsub = redis.pubsub()
    await pubsub.subscribe("options:updates")
    try:
        while True:
            message = await pubsub.get_message(ignore_subscribe_messages=True, timeout=1.0)
            if message and message["type"] == "message":
                await websocket.send_text(message["data"])
            await asyncio.sleep(0.1)
    except WebSocketDisconnect:
        pass
    finally:
        await pubsub.unsubscribe("options:updates")
        await pubsub.close()
```

#### How WebSocket + Redis Pub/Sub Work Together

This is the **real-time data pipeline**. Here's the full chain:

```
Ingestion Service                API Gateway                      Browser
      │                              │                              │
      │  redis.publish(              │                              │
      │    "options:updates",        │                              │
      │    JSON snapshot)            │                              │
      │──────────────────────────►   │                              │
      │                              │  pubsub.get_message()        │
      │                              │──────────────────────►       │
      │                              │                              │
      │                              │  websocket.send_text(data)   │
      │                              │─────────────────────────────►│
      │                              │                              │
```

**Step by step**:
1. **Ingestion service** publishes a JSON snapshot to the Redis channel `options:updates`
2. **API Gateway** has a WebSocket handler that subscribes to that same Redis channel
3. When a message arrives on Redis, the gateway forwards it over the WebSocket to the browser
4. **Frontend** receives the message via `onmessage` callback and updates the UI

**Why Redis pub/sub instead of direct WebSocket from ingestion?**

| Direct approach | Redis pub/sub approach |
|----------------|----------------------|
| Ingestion would need to manage WebSocket connections | Ingestion just publishes; doesn't know about browsers |
| Coupling between services | Services are decoupled |
| If ingestion restarts, all WS connections drop | If ingestion restarts, Redis channel persists |
| Can't scale to multiple API gateway instances | Multiple gateways can subscribe to same channel |

**WebSocket lifecycle**:
1. Browser opens `ws://host:8000/ws/options`
2. Server calls `websocket.accept()` — completes the WebSocket handshake
3. Server enters infinite loop, polling Redis every 100ms
4. When browser disconnects, `WebSocketDisconnect` is caught
5. `finally` block cleans up the Redis subscription

### 7. Service Health Aggregator

[**Code: main.py:236-257**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/main.py#L236)

```python
@app.get("/api/services/health")
async def services_health():
    services = {
        "ingestion": "http://ingestion-service:8010/health",
        "quant-engine": "http://quant-engine:8020/health",
        ...
    }
    async with httpx.AsyncClient(timeout=3.0) as client:
        for name, url in services.items():
            try:
                resp = await client.get(url)
                results[name] = "healthy" if resp.status_code == 200 else "unhealthy"
            except Exception:
                results[name] = "unavailable"
```

**How Docker DNS works here**: Inside the Docker network, each service is reachable by its compose service name. `http://ingestion-service:8010` resolves to the ingestion container's IP via Docker's built-in DNS.

The 3-second timeout is important — if a service is down, we don't want the health check to hang. `unavailable` means the service didn't respond at all (container not running).

### 8. Agent Config API

[**Code: main.py:207-231**](https://github.com/utkarshmahar/stocks/blob/main/services/api-gateway/main.py#L207)

```python
@app.get("/api/config")      # Read all agent parameters
@app.put("/api/config/{key}")  # Update a specific parameter
```

The `agent_config` table stores key-value pairs as JSONB:
- `risk_tolerance`: "moderate"
- `dte_range`: {"min": 14, "max": 60}
- `delta_range`: {"min": 0.15, "max": 0.35}
- `strategies_enabled`: ["put_credit_spread", "cash_secured_put", ...]
- `max_position_pct`: 5

**Why JSONB?** It allows heterogeneous value types (string, object, array, number) in a single column, and PostgreSQL can index and query inside JSONB values.

**The JSONB gotcha** (from `main.py:214-218`):
```python
val = row[1]
if isinstance(val, str):
    try:
        val = json.loads(val)
    except (json.JSONDecodeError, TypeError):
        pass
```

`asyncpg` (the Postgres driver) sometimes returns JSONB as a Python dict/list directly, and sometimes as a string. This code handles both cases defensively.

## How nginx Routes to the API Gateway

[**Code: frontend/nginx.conf**](https://github.com/utkarshmahar/stocks/blob/main/frontend/nginx.conf)

```nginx
location /api/ {
    proxy_pass http://api-gateway:8000;
}

location /ws/ {
    proxy_pass http://api-gateway:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

**Why the WebSocket headers?** WebSocket connections start as HTTP requests and then "upgrade" to the WebSocket protocol. The `Upgrade` and `Connection` headers tell nginx to pass through the protocol upgrade instead of treating it as a normal HTTP proxy.
