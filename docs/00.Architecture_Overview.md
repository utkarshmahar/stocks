## Data Flow Summary

### Real-Time Options Data (every 30 seconds)
```
Schwab API → Ingestion → InfluxDB (historical) + Redis (real-time)
Redis → API Gateway → WebSocket → Frontend Dashboard
InfluxDB → Quant Engine → Options Agent → Risk Engine → Postgres → API Gateway → Frontend
```

### Portfolio Updates (every 5 minutes)
```
Schwab API → Portfolio Service → Postgres → API Gateway → Frontend
```

### Fundamental Analysis (on-demand)
```
User → Frontend → API Gateway → Fundamental Agent → Claude API → Postgres → Frontend
                                       │
                                  Celery Worker → EDGAR → pgvector
```

## Docker Compose Profiles

Services are split into two groups to save resources on the Raspberry Pi:

| Profile | Services | When to use |
|---------|----------|-------------|
| **default** (no flag) | postgres, redis, ingestion-service, api-gateway, frontend | Core data pipeline + UI |
| **full** (`--profile full`) | All of the above + quant-engine, options-agent, portfolio-service, risk-engine, fundamental-agent, worker | Full platform with AI agents |

Start core only:
```bash
sudo docker compose up -d
```

Start everything:
```bash
sudo docker compose --profile full up -d
```

## Key Design Decisions

### 1. InfluxDB Native (not Docker)
InfluxDB runs natively on the Pi because it performs significantly better without Docker's storage overhead on ARM. It was already installed before Docker was set up.

### 2. Dual Output from Ingestion (InfluxDB + Redis)
- **InfluxDB**: Optimized for time-series queries (historical IV percentile, trend charts)
- **Redis pub/sub**: Low-latency real-time push to WebSocket clients (no query overhead)

### 3. Separation of Quant Engine and AI Agent
The quant engine does all **deterministic** math (IV percentile, skew, Greeks). The AI agent only **reasons** about structured quant output. This prevents the LLM from hallucinating numbers.

### 4. Risk Engine as a Gate
Every AI recommendation passes through the risk engine before being saved to Postgres. The risk engine enforces hard rules (position limits, DTE range, probability thresholds) that the LLM cannot override.

### 5. Premium-Adjusted Cost Basis
The portfolio service tracks option premiums collected against underlying positions, computing an adjusted cost basis:
```
Adjusted Cost = (Shares × Avg Price) - Total Premium Collected
```

### 6. Memory-Constrained Profiles
Each container has a memory limit to prevent OOM on the Raspberry Pi 5 (16GB):
- Worker: 512MB (largest — runs Celery with EDGAR parsing)
- Quant Engine + Fundamental Agent: 384MB each
- Most services: 256MB
- Redis + Risk Engine: 128MB
