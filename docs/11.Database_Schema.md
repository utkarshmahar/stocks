# Database Schema — Knowledge Transfer

## Overview

The platform uses **three data stores**, each optimized for different workloads:

| Store | Technology | Purpose |
|-------|-----------|---------|
| **InfluxDB** | Time-series DB (native on host) | Options ticks, IV, Greeks, stock prices |
| **PostgreSQL** | Relational DB (Docker, pgvector) | Recommendations, positions, audit trail, config |
| **Redis** | In-memory store (Docker) | Pub/sub messaging, Celery task queue |

---

## InfluxDB Schema

InfluxDB doesn't have traditional tables — it uses **measurements** (like tables), **tags** (indexed metadata), and **fields** (data values).

### Measurement: `stock_quote`

Written by: [Ingestion Service](https://github.com/utkarshmahar/stocks/blob/main/services/ingestion-service/main.py#L68)

```
stock_quote,symbol=NVDA price=850.25,bid=850.20,ask=850.30,change=5.25,percent_change=0.62,volume=12345678i,high=855.00,low=842.00,open=845.00 1709568000000000000
```

| Component | Values | Indexed? |
|-----------|--------|----------|
| **Tags** | `symbol` | Yes (fast queries by symbol) |
| **Fields** | `price`, `bid`, `ask`, `change`, `percent_change`, `volume` (int), `high`, `low`, `open` | No |
| **Timestamp** | Nanoseconds since epoch | Yes (time-series ordering) |

### Measurement: `options_data`

Written by: [Ingestion Service](https://github.com/utkarshmahar/stocks/blob/main/services/ingestion-service/main.py#L84)

```
options_data,symbol=NVDA,option_type=CALL,expiration=2025-03-21,strike=850 bid=12.50,ask=12.80,last=12.60,mark=12.65,bid_size=25i,ask_size=30i,volume=1543i,open_interest=8920i,implied_volatility=0.285,delta=0.52,gamma=0.015,theta=-0.85,vega=0.45,rho=0.12,in_the_money=true,days_to_expiration=17i 1709568000000000000
```

| Component | Values | Indexed? |
|-----------|--------|----------|
| **Tags** | `symbol`, `option_type` (CALL/PUT), `expiration`, `strike` | Yes |
| **Fields** | `bid`, `ask`, `last`, `mark`, `bid_size` (int), `ask_size` (int), `volume` (int), `open_interest` (int), `implied_volatility`, `delta`, `gamma`, `theta`, `vega`, `rho`, `in_the_money` (bool), `days_to_expiration` (int) | No |

**Design note**: `strike` is a tag (string, indexed) rather than a field. This allows fast filtering by strike price in Flux queries: `filter(fn: (r) => r.strike == "850")`.

### InfluxDB Configuration

| Setting | Value |
|---------|-------|
| Organization | `options_trading` |
| Bucket | `options_db` |
| Precision | Nanoseconds |
| Retention | Default (infinite) |
| URL | `http://host.docker.internal:8086` (from containers) or `http://localhost:8086` (from host) |

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

Enables the `vector` data type for storing and querying high-dimensional embeddings.

### Table: `watchlist`

```sql
CREATE TABLE watchlist (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL UNIQUE,
    active BOOLEAN DEFAULT TRUE,
    added_at TIMESTAMPTZ DEFAULT NOW()
);

-- Seed data
INSERT INTO watchlist (symbol) VALUES ('PANW'), ('QCOM'), ('CSCO')
ON CONFLICT (symbol) DO NOTHING;
```

**Purpose**: Tracks which stocks the platform monitors. The ingestion service polls `WHERE active = TRUE`.

**Soft delete**: Removing a symbol sets `active = FALSE`. Re-adding upserts with `active = TRUE`.

### Table: `recommendations`

```sql
CREATE TABLE recommendations (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL,
    strategy VARCHAR(50) NOT NULL,
    legs JSONB NOT NULL,
    max_profit NUMERIC(12,2),
    max_loss NUMERIC(12,2),
    probability_estimate NUMERIC(5,4),
    capital_required NUMERIC(12,2),
    reasoning_summary TEXT,
    risk_flags JSONB DEFAULT '[]',
    risk_approved BOOLEAN,
    risk_notes TEXT,
    status VARCHAR(20) DEFAULT 'pending',
    generated_at TIMESTAMPTZ DEFAULT NOW(),
    resolved_at TIMESTAMPTZ,
    outcome JSONB
);

CREATE INDEX idx_recommendations_symbol ON recommendations(symbol);
CREATE INDEX idx_recommendations_status ON recommendations(status);
```

**Purpose**: Stores AI-generated trade recommendations with full audit data.

**Key columns**:
- `legs` (JSONB): Array of option legs (strike, type, expiry, action)
- `risk_flags` (JSONB): Array of risk warnings from the risk engine
- `status`: `pending` → `taken` / `ignored` / `paper_traded`
- `resolved_at`: Timestamp when user acted on the recommendation
- `outcome` (JSONB): Future use — track actual P&L after the trade closes

**NUMERIC precision**:
- `NUMERIC(12,2)`: Up to $9,999,999,999.99 — handles any realistic trade size
- `NUMERIC(5,4)`: Up to 9.9999 — for probability (0.0000 to 1.0000)

### Table: `positions`

```sql
CREATE TABLE positions (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL,
    quantity NUMERIC(12,4) NOT NULL,
    avg_price NUMERIC(12,4) NOT NULL,
    current_price NUMERIC(12,4),
    market_value NUMERIC(14,2),
    cost_basis NUMERIC(14,2),
    pnl NUMERIC(14,2),
    pnl_percent NUMERIC(8,4),
    asset_type VARCHAR(20) DEFAULT 'EQUITY',
    last_synced TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(symbol, asset_type)
);
```

**Purpose**: Synced from Schwab every 5 minutes by the portfolio service.

**`UNIQUE(symbol, asset_type)`**: Allows the same symbol to exist as both EQUITY and OPTION positions without conflict.

**`NUMERIC(12,4)`**: 4 decimal places for fractional shares and precise pricing.

### Table: `premium_collections`

```sql
CREATE TABLE premium_collections (
    id SERIAL PRIMARY KEY,
    underlying_symbol VARCHAR(10) NOT NULL,
    option_symbol VARCHAR(50),
    strategy VARCHAR(50),
    premium_collected NUMERIC(12,2) NOT NULL,
    contracts INTEGER DEFAULT 1,
    open_date TIMESTAMPTZ NOT NULL,
    close_date TIMESTAMPTZ,
    status VARCHAR(20) DEFAULT 'open',
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_premium_underlying ON premium_collections(underlying_symbol);
```

**Purpose**: Tracks premium collected from sold options, used to calculate adjusted cost basis.

**Status flow**: `open` → `closed` / `expired` / `assigned`

### Table: `fundamental_reports`

```sql
CREATE TABLE fundamental_reports (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL,
    report_type VARCHAR(30) NOT NULL,
    report_data JSONB NOT NULL,
    ai_narrative TEXT,
    intrinsic_value NUMERIC(12,2),
    current_price NUMERIC(12,2),
    upside_percent NUMERIC(8,2),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_reports_symbol ON fundamental_reports(symbol);
```

**Purpose**: Stores AI-generated fundamental analysis reports.

**`report_data` (JSONB)**: Contains the full structured report (thesis, revenue growth array, EPS array, margins, moat assessment). Using JSONB allows the frontend to render charts from this data.

### Table: `filing_embeddings`

```sql
CREATE TABLE filing_embeddings (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL,
    filing_type VARCHAR(10) NOT NULL,
    filing_date DATE NOT NULL,
    chunk_index INTEGER NOT NULL,
    chunk_text TEXT NOT NULL,
    embedding vector(1024),
    metadata JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_filing_symbol ON filing_embeddings(symbol);
CREATE INDEX idx_filing_embedding ON filing_embeddings USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
```

**Purpose**: Stores SEC filing text chunks for RAG (Retrieval-Augmented Generation).

**`embedding vector(1024)`**: A 1024-dimensional vector (pgvector type). This stores the semantic embedding of each text chunk.

**IVFFlat index**: An approximate nearest-neighbor index for fast vector similarity search. `lists = 100` creates 100 clusters for the index — queries search the nearest clusters rather than scanning all vectors.

**`vector_cosine_ops`**: Uses cosine similarity for distance calculation (standard for text embeddings).

### Table: `audit_log`

```sql
CREATE TABLE audit_log (
    id SERIAL PRIMARY KEY,
    service VARCHAR(50) NOT NULL,
    action VARCHAR(100) NOT NULL,
    details JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_audit_service ON audit_log(service);
CREATE INDEX idx_audit_created ON audit_log(created_at);
```

**Purpose**: Every significant action across all services is logged here. This provides a complete history of what the system did and when.

**Example entries**:
```json
{"service": "options-agent", "action": "recommendation_generated", "details": {"symbol": "NVDA", "strategy": "Put Credit Spread"}}
```

### Table: `agent_config`

```sql
CREATE TABLE agent_config (
    id SERIAL PRIMARY KEY,
    key VARCHAR(100) NOT NULL UNIQUE,
    value JSONB NOT NULL,
    description TEXT,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Seed data
INSERT INTO agent_config (key, value, description) VALUES
    ('risk_tolerance', '"moderate"', 'Risk tolerance: conservative, moderate, aggressive'),
    ('dte_range', '{"min": 14, "max": 60}', 'Days to expiration range'),
    ('delta_range', '{"min": 0.15, "max": 0.35}', 'Delta range for selling'),
    ('strategies_enabled', '["put_credit_spread", "cash_secured_put", "covered_call", "iron_condor"]', 'Enabled strategies'),
    ('max_position_pct', '5', 'Max position size %'),
    ('max_sector_pct', '20', 'Max sector exposure %')
ON CONFLICT (key) DO NOTHING;
```

**Purpose**: Key-value store for agent parameters, editable from the Settings page.

**Why JSONB values?** Different config keys have different value types (string, object, array, number). JSONB handles all of them in a single column.

## Redis Data

Redis doesn't have a schema, but here's what's stored:

### Pub/Sub Channels

| Channel | Publisher | Subscriber | Data |
|---------|-----------|------------|------|
| `options:updates` | Ingestion Service | API Gateway | JSON options snapshot per symbol |
| `portfolio:updates` | Portfolio Service | (future) API Gateway | Position sync events |

### Celery Queues

| Key pattern | Purpose |
|-------------|---------|
| `celery` (list) | Default task queue |
| `celery-task-meta-*` | Task result storage |

## Entity Relationship Diagram

```
watchlist (1) ───── (*) ingestion polls
    │
    └─── symbol ──── (*) recommendations
                          │
                          └─── risk_flags, risk_approved
                          └─── legs (JSONB)

positions (1) ───── (*) premium_collections
    │                       │
    └─── symbol ────────── underlying_symbol
    └─── cost_basis - SUM(premium) = adjusted_cost

fundamental_reports ◄──── filing_embeddings
    │                       │
    └─── symbol ────────── symbol
    └─── report_data        └─── chunk_text + embedding

agent_config (key-value pairs read by all services)

audit_log (append-only log of all actions)
```
