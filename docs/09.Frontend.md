# Frontend — Knowledge Transfer

## What is it?

A **single-page application** (SPA) built with React 18, Vite 6, and TailwindCSS. It provides 5 pages that visualize all data from the backend services: live options chains, AI recommendations, fundamental analysis, portfolio P&L, and system settings.

## Technology Deep Dives

### React (Component-Based UI)

React renders the UI as a tree of **components** — reusable, self-contained pieces of UI. Each component:
- Takes **props** (inputs)
- Manages **state** (internal data)
- Returns **JSX** (HTML-like syntax)

This project uses **functional components** with **hooks** (no class components):
```tsx
export default function Dashboard() {
  const [selectedSymbol, setSelectedSymbol] = useState('PANW')
  // ... render UI
}
```

### Vite (Build Tool)

[**Code: vite.config.ts**](https://github.com/utkarshmahar/stocks/blob/main/frontend/vite.config.ts)

```typescript
export default defineConfig({
  plugins: [react()],
  server: {
    host: '0.0.0.0',
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8000',
      '/ws': { target: 'ws://localhost:8000', ws: true },
    },
  },
})
```

**What is Vite?** A modern build tool that's 10-100x faster than Webpack:
- **Dev mode**: Uses native ES modules in the browser (no bundling step)
- **Build mode**: Uses Rollup for optimized production bundles
- **HMR** (Hot Module Replacement): Changes appear instantly without full page reload

**Dev proxy**: During development (`npm run dev`), Vite proxies `/api/*` to `http://localhost:8000` (the API gateway). This avoids CORS issues during development. In production, nginx handles the proxying.

### TailwindCSS (Utility-First CSS)

Instead of writing CSS classes like `.card-header { padding: 1rem; font-weight: bold; }`, Tailwind uses atomic utility classes directly in HTML:

```tsx
<div className="bg-gray-900 rounded-lg p-6 border border-gray-800">
```

| Class | CSS equivalent |
|-------|---------------|
| `bg-gray-900` | `background-color: #111827` |
| `rounded-lg` | `border-radius: 0.5rem` |
| `p-6` | `padding: 1.5rem` |
| `border` | `border-width: 1px` |
| `border-gray-800` | `border-color: #1f2937` |
| `text-emerald-400` | `color: #34d399` |
| `text-2xl` | `font-size: 1.5rem` |
| `font-bold` | `font-weight: 700` |

**Benefits**: No CSS file management, responsive design built-in (`md:grid-cols-2`), dark mode support, tiny production bundle (unused classes are purged).

### TanStack Query (Server State Management)

[**Code: Dashboard.tsx:45-59**](https://github.com/utkarshmahar/stocks/blob/main/frontend/src/pages/Dashboard.tsx#L45)

```tsx
const { data: quote } = useQuery({
  queryKey: ['quote', selectedSymbol],
  queryFn: () => apiFetch<Quote>(`/quote/${selectedSymbol}`),
  refetchInterval: 15000,
})
```

**What is TanStack Query?** A library that manages the entire lifecycle of fetching data from APIs:

| Feature | What it does |
|---------|-------------|
| **Caching** | Stores API responses; subsequent renders use cache |
| **Auto-refetch** | `refetchInterval: 15000` re-fetches every 15 seconds |
| **Loading states** | `isLoading` is true until data arrives |
| **Error handling** | `isError` + `error` for failed requests |
| **Cache invalidation** | `queryClient.invalidateQueries()` forces refetch |
| **Deduplication** | Multiple components using same query key share one request |

**`queryKey`**: A unique identifier for each query. `['quote', 'NVDA']` and `['quote', 'PANW']` are cached separately.

**Mutations** (for POST/PUT/DELETE):
```tsx
const updateStatus = useMutation({
  mutationFn: ({ id, status }) => apiFetch(`/recommendations/${id}`, {
    method: 'PATCH',
    body: JSON.stringify({ status }),
  }),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ['recommendations'] }),
})
```

After a successful mutation, `invalidateQueries` forces the recommendations list to refetch with the updated data.

## Hooks (Shared Logic)

### useApi

[**Code: hooks/useApi.ts**](https://github.com/utkarshmahar/stocks/blob/main/frontend/src/hooks/useApi.ts)

```typescript
const API_BASE = '/api'

export async function apiFetch<T>(path: string, options?: RequestInit): Promise<T> {
  const resp = await fetch(`${API_BASE}${path}`, {
    headers: { 'Content-Type': 'application/json' },
    ...options,
  })
  if (!resp.ok) throw new Error(`API error: ${resp.status} ${resp.statusText}`)
  return resp.json()
}
```

A thin wrapper around `fetch()` that:
- Prepends `/api` to all paths
- Sets `Content-Type: application/json` header
- Throws on non-200 responses (TanStack Query catches these)
- Returns typed JSON via generics (`apiFetch<Quote>(...)` returns `Promise<Quote>`)

### useWebSocket

[**Code: hooks/useWebSocket.ts**](https://github.com/utkarshmahar/stocks/blob/main/frontend/src/hooks/useWebSocket.ts)

```typescript
export function useWebSocket(url: string) {
  const [lastMessage, setLastMessage] = useState<any>(null)
  const [connected, setConnected] = useState(false)

  const connect = useCallback(() => {
    const ws = new WebSocket(url)
    ws.onopen = () => setConnected(true)
    ws.onclose = () => {
      setConnected(false)
      setTimeout(connect, 3000)  // Auto-reconnect after 3 seconds
    }
    ws.onmessage = (event) => setLastMessage(JSON.parse(event.data))
  }, [url])

  useEffect(() => { connect(); return () => wsRef.current?.close() }, [connect])

  return { lastMessage, connected }
}
```

**Key features**:
- **Auto-reconnect**: If the WebSocket drops, it reconnects after 3 seconds
- **Parsed messages**: Incoming JSON is parsed automatically
- **Connection status**: `connected` boolean drives the green/red indicator in the UI
- **Cleanup**: WebSocket is closed when the component unmounts

**How it's used in Dashboard**:
```tsx
const wsUrl = `ws://${window.location.hostname}:8000/ws/options`
const { lastMessage, connected } = useWebSocket(wsUrl)

useEffect(() => {
  if (lastMessage?.symbol === selectedSymbol && lastMessage?.quote?.price) {
    setPriceHistory(prev => [...prev, { time: new Date().toLocaleTimeString(), price: lastMessage.quote.price }].slice(-60))
  }
}, [lastMessage, selectedSymbol])
```

Every time a WebSocket message arrives for the selected symbol, the price is appended to a 60-point history for the real-time chart.

## Production Build & Deployment

### Build Process

[**Code: Dockerfile**](https://github.com/utkarshmahar/stocks/blob/main/frontend/Dockerfile)

```dockerfile
# Stage 1: Build
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json ./
RUN npm install
COPY . .
RUN npm run build   # tsc + vite build → /app/dist/

# Stage 2: Serve
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

**Multi-stage Docker build**:
1. **Build stage**: Node.js compiles TypeScript + bundles React with Vite → static HTML/JS/CSS in `/app/dist/`
2. **Serve stage**: nginx serves the static files. The Node.js runtime is discarded — the final image is tiny (~30MB).

### nginx Configuration

[**Code: nginx.conf**](https://github.com/utkarshmahar/stocks/blob/main/frontend/nginx.conf)

```nginx
location / {
    try_files $uri $uri/ /index.html;    # SPA fallback
}
location /api/ {
    proxy_pass http://api-gateway:8000;   # REST proxy
}
location /ws/ {
    proxy_pass http://api-gateway:8000;   # WebSocket proxy
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

**`try_files $uri $uri/ /index.html`**: This is the SPA fallback. When you navigate to `/portfolio`, nginx doesn't have a `/portfolio` file — it serves `index.html`, and React Router handles the routing in the browser.

**WebSocket proxy headers**: `Upgrade` and `Connection` headers are required for the HTTP → WebSocket protocol upgrade to pass through nginx.
