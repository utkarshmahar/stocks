# Fundamental Agent — Knowledge Transfer

## What is it?

The Fundamental Agent performs **long-term stock valuation analysis** using a combination of:
1. **EDGAR RAG** (Retrieval-Augmented Generation) — searches SEC filing excerpts stored in pgvector
2. **DCF Model** (Discounted Cash Flow) — deterministic intrinsic value computation
3. **Claude AI** — generates a structured analysis report with thesis, risks, and charts

This is the **long-term counterpart** to the Options AI Agent (which focuses on short-term income trades).

---

## High-Level Flow

```
User (via frontend)
    │
    │ POST /api/fundamental/NVDA/analyze
    │
    ▼
Fundamental Agent
    │
    ├─ 1. Search pgvector for SEC filing excerpts
    ├─ 2. Get current price from API Gateway
    ├─ 3. Build prompt with filing context + price
    ├─ 4. Call Claude Sonnet API
    ├─ 5. Parse JSON report
    ├─ 6. Optionally run DCF model
    ├─ 7. Save report to Postgres
    │
    ▼
Frontend displays: thesis, valuation, charts, DCF
```

## Key Components

### 1. EDGAR Filing Search (RAG)

[**Code: main.py:73-91**](https://github.com/utkarshmahar/stocks/blob/main/services/fundamental-agent/main.py#L73)

```python
async def search_filings(symbol: str, query_text: str, limit: int = 5) -> list[dict]:
    factory = get_session_factory()
    async with factory() as session:
        result = await session.execute(
            text("""
                SELECT chunk_text, filing_type, filing_date, metadata
                FROM filing_embeddings
                WHERE symbol = :s
                ORDER BY filing_date DESC
                LIMIT :lim
            """),
            {"s": symbol.upper(), "lim": limit},
        )
```

#### What is RAG?

**Retrieval-Augmented Generation** is a pattern where:
1. **Retrieval**: Search a database for relevant documents (here: SEC filing chunks)
2. **Augmentation**: Include those documents in the LLM prompt as context
3. **Generation**: The LLM generates a response grounded in the retrieved data

**Without RAG**: The LLM would hallucinate financial numbers from its training data (which may be outdated).

**With RAG**: The LLM reads actual 10-K/10-Q excerpts and bases its analysis on real numbers.

#### What is pgvector?

[pgvector](https://github.com/pgvector/pgvector) is a PostgreSQL extension that adds **vector similarity search**. It enables:

- Storing high-dimensional vectors (embeddings) alongside regular data
- Searching for "semantically similar" text using cosine similarity
- Indexing with IVFFlat for fast approximate nearest-neighbor search

**The `filing_embeddings` table** stores:
- `chunk_text`: A ~1000-word excerpt from a SEC filing
- `embedding`: A 1024-dimensional vector representation of that text
- `filing_type`: "10-K" (annual) or "10-Q" (quarterly)
- `filing_date`: When the filing was published

**Current implementation note**: The current code uses `ORDER BY filing_date DESC` (most recent first) rather than vector similarity search. This is a simplified version — full vector search would use:
```sql
SELECT * FROM filing_embeddings
ORDER BY embedding <=> query_embedding
LIMIT 5;
```
This would find the 5 most semantically similar chunks to the query, regardless of date.

### 2. DCF Model (Discounted Cash Flow)

[**Code: main.py:38-70**](https://github.com/utkarshmahar/stocks/blob/main/services/fundamental-agent/main.py#L38)

```python
def compute_dcf(fcf_values: list[float], growth_rate: float = 0.08,
                discount_rate: float = 0.10, terminal_growth: float = 0.03,
                shares_outstanding: float = 1.0) -> dict:
```

#### What is DCF?

DCF is the **gold standard** of stock valuation. It computes what a company is worth today based on its future cash flows:

```
Intrinsic Value = (Present Value of Future Cash Flows) + (Present Value of Terminal Value)
                  ─────────────────────────────────────────────────────────────────────
                                          Shares Outstanding
```

**Step by step**:

**Step 1: Project future cash flows** (5 years)
```python
for i in range(1, 6):
    proj = last_fcf * ((1 + growth_rate) ** i)
    projected.append(proj)
```
Example: If last year's FCF was $10B and growth = 8%:
- Year 1: $10B × 1.08 = $10.8B
- Year 2: $10B × 1.08² = $11.66B
- Year 3: $10B × 1.08³ = $12.60B
- Year 4: $10B × 1.08⁴ = $13.60B
- Year 5: $10B × 1.08⁵ = $14.69B

**Step 2: Calculate terminal value** (company's value from year 6 to infinity)
```python
terminal_fcf = projected[-1] * (1 + terminal_growth)
terminal_value = terminal_fcf / (discount_rate - terminal_growth)
```
Using the **Gordon Growth Model**: Terminal Value = FCF₆ / (r - g)
- FCF₆ = $14.69B × 1.03 = $15.13B
- Terminal Value = $15.13B / (0.10 - 0.03) = **$216.1B**

**Step 3: Discount everything to present value**
```python
pv_fcfs = sum(fcf / ((1 + discount_rate) ** (i + 1)) for i, fcf in enumerate(projected))
pv_terminal = terminal_value / ((1 + discount_rate) ** 5)
enterprise_value = pv_fcfs + pv_terminal
```

Future dollars are worth less than today's dollars (time value of money). Divide each cash flow by (1 + discount_rate)^years.

**Step 4: Per-share value**
```python
intrinsic_per_share = enterprise_value / shares_outstanding
```

**Key assumptions** (user-configurable):
| Parameter | Default | Meaning |
|-----------|---------|---------|
| `growth_rate` | 8% | How fast FCF grows annually |
| `discount_rate` | 10% | Required rate of return (WACC proxy) |
| `terminal_growth` | 3% | Perpetual growth rate (GDP-like) |
| `shares_outstanding` | 1.0 | Total shares (must be provided) |

```python
await session.execute(text("""
    INSERT INTO fundamental_reports
    (symbol, report_type, report_data, ai_narrative, intrinsic_value, current_price, upside_percent)
    VALUES (:s, :rt, :rd, :ai, :iv, :cp, :up)
"""), {
    "s": symbol,
    "rt": "full_analysis",
    "rd": json.dumps(report_data),
    "ai": report_data.get("thesis", ""),
    "iv": intrinsic_value,  # From DCF
    "cp": current_price,
    "up": upside_percent,   # ((intrinsic / current) - 1) × 100
})
```

The full report (JSON) is stored in the `report_data` JSONB column. Key metrics are also stored in dedicated columns for easy querying.
