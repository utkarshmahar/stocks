# Portfolio Service — Knowledge Transfer

## What is it?

The Portfolio Service manages **real-time portfolio tracking** with a unique feature: **premium-adjusted cost basis**. When you sell options against a stock position, the collected premium reduces your effective cost. This service tracks that adjustment automatically.

---

## High-Level Flow

```
Schwab API (positions)
    │
    ▼
Portfolio Service
    │
    ├─ Fetch positions via Schwab SDK
    ├─ Calculate P&L
    ├─ UPSERT to Postgres
    └─ Publish update to Redis
    │
    ▼
API Gateway → Frontend (Portfolio page)
```

---

## Key Concept: Premium-Adjusted Cost Basis

This is the core financial concept of this service:

```
Adjusted Cost Basis = (Shares × Avg Price) - Total Premium Collected
Effective Cost/Share = Adjusted Cost Basis / Shares
Adjusted P&L = Market Value - Adjusted Cost Basis
```

**Example**:
- You buy 100 shares of NVDA at $850 → Cost basis = $85,000
- You sell a covered call and collect $420 premium → Total premium = $420
- You sell another covered call, collect $380 → Total premium = $800
- Adjusted cost basis = $85,000 - $800 = **$84,200**
- Effective cost per share = $84,200 / 100 = **$842.00** (vs original $850)
- If NVDA is now at $860: Adjusted P&L = ($860 × 100) - $84,200 = **$1,800** (vs $1,000 without premium)

The premium collected acts as a **cost reducer**, making your position more profitable even if the stock doesn't move.

### 1. Schwab Position Fetching

[**Code: main.py:22-58**](https://github.com/utkarshmahar/stocks/blob/main/services/portfolio-service/main.py#L22)

```python
def fetch_schwab_positions(client) -> list[dict]:
    # Step 1: Get account numbers/hashes
    acct_resp = client.get_account_numbers()
    accounts = acct_resp.json()

    # Step 2: Get positions for first account
    hash_val = accounts[0].get("hashValue", "")
    response = client.get_account(hash_val, fields=[client.Account.Fields.POSITIONS])

    # Step 3: Parse positions
    for pos in acct_data.get("positions", []):
        positions.append({
            "symbol": instrument.get("symbol", ""),
            "quantity": pos.get("longQuantity", 0) - pos.get("shortQuantity", 0),
            "avg_price": pos.get("averagePrice", 0),
            "current_price": pos.get("marketValue", 0) / max(pos.get("longQuantity", 1), 1),
            "market_value": pos.get("marketValue", 0),
            "asset_type": instrument.get("assetType", "EQUITY"),
        })
```

**Schwab API quirk**: You can't call the positions endpoint directly. You must first call `get_account_numbers()` to get a `hashValue`, then use that hash to fetch account details. This is a security pattern — the account number itself is never exposed.

**Net quantity**: `longQuantity - shortQuantity` gives the net position. If you're long 100 shares and short 50 (via lending), net = 50.

**Current price derivation**: `marketValue / longQuantity` is an approximation. For exact price, you'd query the quote API separately.

### 2. Position Sync to Postgres

[**Code: main.py:61-101**](https://github.com/utkarshmahar/stocks/blob/main/services/portfolio-service/main.py#L61)

```python
async def sync_positions():
    client = get_schwab_client()
    positions = fetch_schwab_positions(client)

    for pos in positions:
        cost_basis = pos["quantity"] * pos["avg_price"]
        pnl = pos["market_value"] - cost_basis
        pnl_pct = (pnl / cost_basis * 100) if cost_basis > 0 else 0

        await session.execute(text("""
            INSERT INTO positions (...) VALUES (...)
            ON CONFLICT (symbol, asset_type)
            DO UPDATE SET quantity = :qty, avg_price = :avg, ...
        """), {...})

    # Publish update to Redis
    await redis.publish("portfolio:updates", json.dumps({
        "event": "positions_synced",
        "count": len(positions),
        "timestamp": datetime.utcnow().isoformat(),
    }))
```

**UPSERT pattern** (`ON CONFLICT ... DO UPDATE`): If a position for that symbol+asset_type already exists, update it. Otherwise, insert a new row. This prevents duplicate rows and ensures the latest data.

**Redis publish**: After syncing, publishes an event so the frontend can show a "positions updated" notification via WebSocket.

### 3. Adjusted Cost Basis Endpoint

[**Code: main.py:110-144**](https://github.com/utkarshmahar/stocks/blob/main/services/portfolio-service/main.py#L110)

```python
@app.get("/api/portfolio/adjusted")
async def get_adjusted_positions(db: AsyncSession = Depends(get_db)):
    result = await db.execute(text("""
        SELECT p.*, COALESCE(SUM(pc.premium_collected * pc.contracts * 100), 0) as total_premium
        FROM positions p
        LEFT JOIN premium_collections pc ON p.symbol = pc.underlying_symbol
            AND pc.status != 'cancelled'
        GROUP BY p.id
    """))
```

**SQL breakdown**:
- `LEFT JOIN premium_collections`: Join positions with their associated premium records
- `COALESCE(SUM(...), 0)`: Sum all premiums; default to 0 if no premiums exist
- `pc.premium_collected * pc.contracts * 100`: Premium per contract × number of contracts × 100 (each contract = 100 shares)
- `pc.status != 'cancelled'`: Exclude cancelled premiums
- `GROUP BY p.id`: Aggregate premiums per position

**Why `* 100`?** Options contracts represent 100 shares. If you sell 1 contract at $4.20 premium, you collect $420.

### 5. Background Sync Loop

[**Code: main.py:170-182**](https://github.com/utkarshmahar/stocks/blob/main/services/portfolio-service/main.py#L170)

```python
async def sync_loop():
    while True:
        try:
            await sync_positions()
        except Exception as e:
            print(f"[PORTFOLIO] Sync error: {e}")
        await asyncio.sleep(300)  # 5 minutes

@app.on_event("startup")
async def startup():
    asyncio.create_task(sync_loop())
```

Positions are synced from Schwab every 5 minutes. This runs as an `asyncio` background task — same pattern as the ingestion service.

**Why 5 minutes?** Schwab's API has rate limits, and portfolio positions don't change second-by-second. 5 minutes is a good balance.

## Technology Deep Dive: schwab-py SDK

The `schwab-py` library wraps the Schwab (formerly TD Ameritrade) broker API:

```python
from schwab.client import Client
import schwab.auth

# Authentication: reads from token file (OAuth2 refresh token)
client = schwab.auth.client_from_token_file(token_path, api_key, app_secret)

# Account API
client.get_account_numbers()  → [{"accountNumber": "...", "hashValue": "..."}]
client.get_account(hash_val, fields=[Client.Account.Fields.POSITIONS])  → account + positions

# Market Data API
client.get_option_chain(symbol, ...)  → full options chain
client.get_quote(symbol)  → real-time stock quote
```

**OAuth2 flow**:
1. Initial auth: User runs `authenticate_schwab.py` which opens a browser for Schwab login
2. After login, Schwab redirects with an auth code
3. The script exchanges the code for access + refresh tokens
4. Tokens are saved to `schwab_token.json` (mounted into Docker containers)
5. The refresh token lasts 7 days — after that, re-authentication is needed
6. `schwab-py` handles token refresh automatically within the 7-day window
